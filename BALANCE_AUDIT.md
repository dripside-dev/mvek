## 3. Шансы (вероятности)

- 🟡 **Удача (luck) НЕ влияет на редкость предметов** — `mvek/items/items.py:976-990`. `_weighted_pick`/`random_pickup`/`random_shop_item` не принимают luck — пул одинаков при luck=0 и luck=4. При этом 5 предметов + трансформация hacker дают luck, старты до luck=4 (`student.py:157`), UI рисует «УДАЧА» (`main.py:1493`). luck влияет ТОЛЬКО на таблицу дропов комнаты (`room.py:161-181`). Скрытое несоответствие ожиданиям (как в TBoI).
  - **Альтернатива баланса**: архитектурная проблема — `floor.py` генерирует пикапы без ссылки на игрока. Практичнее выбирать конкретный ItemPickup в момент подбора/входа в комнату (player доступен) и домножать вес тиров 3-4 на `(1+luck*коэф)`; либо хранить snapshot luck на Floor. Коэффициент подбирать с учётом весов {3:20,4:6}.

- 🟡 **Легендарки (tier-4) выпадают с шансом 0.23% за ролл без pity** — `mvek/items/items.py:911, 976-990`. `TIER_WEIGHTS {1:100,2:50,3:20,4:6}`; 55 предметов (tier1=11, tier2=22, tier3=18, tier4=4), сумма весов = **2584** (не 2566 — арифметическая опечатка автора, проценты верны). Конкретная легендарка = 6/2584 = 0.232%, весь пул = 0.93%. Pity/bad-luck protection отсутствует.
  - **Альтернатива баланса**: поднять вес tier4 (6→15: per-item ~0.58%, пул ~2.3%) и/или ввести pity-счётчик:
    ```python
    def _weighted_pick(rng, pool, pity=0):
        weights = []
        for it in pool:
            t = it.get('tier', _DEFAULT_TIER)
            w = TIER_WEIGHTS.get(t, 1)
            if t == 4:
                w += pity * 3
            weights.append(w)
        return rng.choices(pool, weights=weights, k=1)[0]
    ```
    Хранить `rolls_since_legendary` на прогоне, сбрасывать при tier4. Обновить вызовы `random_pickup`/`random_shop_item`.

- 🟡 **Луч-уклон удачи смещает ВСЕ пороги дроп-таблицы, а не только хорошие** — `mvek/world/room.py:161-181`. `roll -= luck*0.05` глобально сдвигает влево. При luck=2 «пусто» падает 15%→5%, ключ 10%→20%; при luck=4 «пусто» исчезает (P=0), отрицательный roll делает ключ почти гарантированным. Уточнение: монета НЕ растёт (интервал шириной 0.45 стабилен), сердце НЕ немонотонно (~15% постоянно) — реальный вред в инфляции ключа и исчезновении «пусто».
  - **Альтернатива баланса**: оставить основную таблицу фиксированной (`roll = random.random()` без вычитания), удачу — отдельными бросками на бонусы (частично уже есть, строка 179). Если нужен общий уклон: `roll = max(0.0, random.random() - luck*0.02)` с пересчётом порогов, сохраняя «пусто» ≥5%. Ограничить `min(luck, cap)`.

- 🟡 **Бонус-ключ масштабируется удачей без капа** — `mvek/world/room.py:179-181`. `bonus_chance = 0.18 + max(0, luck)*0.05` без верхнего предела.
