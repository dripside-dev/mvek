# MVEK — Дизайн-документ для ИИ-художника

> Этот документ — единый источник правды по визуальному стилю игры **MVEK**.
> Его задача — дать ИИ-дизайнеру (или человеку-художнику) всё необходимое,
> чтобы отрисовать **иконки предметов** и **общий арт игры** в едином стиле,
> согласованном с уже существующим кодом и палитрой.

---

## 1. Что такое MVEK (контекст за 30 секунд)

**MVEK** — top-down рогалик в духе *The Binding of Isaac*, переосмысленный в
антураже российского студенческого колледжа. Игрок управляет студентом,
проходит 5 этажей учебного корпуса, отбивается от тьюторов и преподавателей,
собирает предметы и в конце спускается к Директору.

**Жанр:** top-down twin-stick shooter / roguelite
**Движок:** Python + pygame, окно **960×720**, 60 FPS
**Сеттинг:** учебный корпус колледжа — вестибюль, коридоры, кафедры,
лаборатории, деканат.

### Ключевые замены тематики (важно для метафор в артах)
| В классическом рогалике | В MVEK |
|---|---|
| Кровь / демоны / трэш | **Ничего этого нет.** Стиль семейный |
| Снаряды / слёзы | **Доклады, рефераты, билеты** (листы бумаги) |
| Деньги / монеты | **Рубли** |
| Здоровье (сердца) | Шкала **«Любовь и одобрение»** |
| Синие (soul) сердца | **Зачётные** синие сердца |
| Боссы-монстры | **Тьюторы, преподаватели, Директор** |

> **Правило №1: никакой крови, демонов, насилия, неуместных шуток.**
> Всё дружелюбное, ламповое, узнаваемо «студенческое».

## 2. Общий визуальный стиль

- **Пиксель-арт**, чёткий читаемый силуэт, контурная обводка тёмным для контраста.
- **Ретро game-UI**: тёмные панели, чёткие рамки, лёгкие скан-линии.
- Тёплая фиолетово-коричневая основа окружения + яркие акценты на важном
  (игрок — голубой, бумага — кремовая, валюта/легендарки — золото, угроза — красно-розовая).
- Освещение «вечерний корпус»: тёмный фон, мягкое свечение источников света.
- Шрифты UI: **Pixelify Sans** (заголовки/имена), **JetBrains Mono** (тех-текст),
  **Handjet** (крупный дисплейный). Все — с поддержкой кириллицы.

## 4. Система тиров (редкость предметов)

В игре **4 тира**. Чем выше тир — тем реже выпадает предмет и тем дороже он в
магазине. Редкость **обязательно** отражается визуально (аура/рамка/свечение).

| Тир | Название | Аура / рамка | Вес дропа | Цена в магазине | Кол-во |
|---|---|---|---|---|---|
| **Т1** | Обычные | Серая `#9A9AA6`, без свечения | 100 | 7 ₽ | 11 |
| **Т2** | Крепкие | Зелёная `#56C878`, мягкое свечение | 50 | 12 ₽ | 22 |
| **Т3** | Сильные | Фиолетовая `#C878FF`, пульсация | 20 | 20 ₽ | 18 |
| **Т4** | Легендарные | Золотая `#F0C850`, пульсация + искры | 6 | 35 ₽ | 4 |

**Как тир влияет на арт иконки:**
- **Т1** — простой силуэт, минимум деталей, спокойные цвета.
- **Т2** — чуть больше деталей, аккуратный блик.
- **Т3** — заметные детали, насыщенный цвет, можно лёгкое свечение элемента.
- **Т4** — максимум проработки, металлический блеск, золотые акценты, «дорогой» вид.

> Аура/рамка/искры — это **UI-обрамление слота** (см. готовый каталог
> `site/pages/MVEK - Каталог предметов.html`), а не часть самого спрайта.
> Сам спрайт рисуется на **прозрачном фоне** в пределах 28×28.

## 5. Технические требования к иконкам предметов

| Параметр | Значение |
|---|---|
| **Размер спрайта** | **28×28 px** |
| **Полезная зона силуэта** | в пределах **±14 px** от центра |
| **Фон** | прозрачный (alpha) |
| **Композиция** | один предмет, по центру |
| **Стиль** | пиксель-арт, тёмная контурная обводка |
| **Палитра** | ограниченная, из раздела 3 |
| **Масштаб в UI** | спрайт кратно увеличивается (nearest-neighbor), поэтому **без сглаживания** |

### Как иконки устроены в коде (для справки)
Сейчас иконки описаны мини-языком примитивов в `mvek/items/items.py`
(функция `paint_icon`). Опкоды:

| Опкод | Параметры | Рисует |
|---|---|---|
| `c` | `color, dx, dy, r` | залитый круг |
| `o` | `color, dx, dy, r, w` | контурный круг (толщина `w`) |
| `r` | `color, dx, dy, w, h` | залитый прямоугольник (центр в `dx,dy`) |
| `R` | `color, dx, dy, w, h, lw` | контурный прямоугольник |
| `p` | `color, [(x,y), …]` | залитый полигон |
| `l` | `color, x1, y1, x2, y2, w` | линия |

Координаты — пиксели от центра при `scale=1.0`. Пример (стопка денег
«Стипендия», Т4):

```python
"money_stack": [
    ('r', (120, 180, 120), 0, 0, 22, 14),     # зелёная пачка
    ('R', (60, 120, 60), 0, 0, 22, 14, 1),     # тёмный контур
    ('c', (240, 220, 100), 0, 0, 5),           # золотая монета по центру
    ('c', (60, 120, 60), 0, 0, 5),             # ободок монеты
    ('l', (60, 120, 60), -8, -4, -8, 4, 1),    # перевязка слева
    ('l', (60, 120, 60), 8, -4, 8, 4, 1),      # перевязка справа
],
```

> **Художнику не обязательно** работать в этом мини-языке. Можно отдать
> готовые **PNG 28×28 с прозрачностью** — мы вставим их вместо примитивов.
> Главное — соблюсти размер, палитру и стиль.

  синие зачётные `#B4C8FF`.
- Счётчики: рубли (золото `#F0C850`), ключи, бомбы (хлопушки).
- Слот активного предмета + шкала кулдауна.
- Миникарта: метки комнат разных типов.

## 8. Чек-лист сдачи (для художника)

- [ ] Спрайт **28×28 px**, силуёт в **±14 px**, **прозрачный фон**.
- [ ] Только цвета из палитры (раздел 3), тёмная контурная обводка.
- [ ] **Без сглаживания** (чистый пиксель-арт, nearest-neighbor).
- [ ] Силуэт читается на маленьком размере и на тёмном фоне.
- [ ] Уровень детализации соответствует тиру (Т1 проще → Т4 богаче).
- [ ] **Семейный стиль**: ни крови, ни трэша, ни неуместных шуток.
- [ ] Имя файла = латиницей по смыслу (напр. `money_stack.png`), 1 файл = 1 предмет.
