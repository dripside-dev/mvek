# MVEK — рогалик про студенческие будни

Top‑down рогалик в духе классики жанра, переосмысленный в антураже
студенческого колледжа МВЭК. Игрок управляет студентом, проходит
этажи учебного корпуса, отбивается от тьюторов и преподавателей,
собирает предметы и в конце спускается к Директору.

> **Стиль семейный**: никакой крови, никаких демонов, никаких какашек.
> Снаряды — рефераты и билеты, валюта — рубли, «здоровье» — шкала
> «Любовь и одобрение».

## 1. Быстрый старт

### 1.1. Запуск

Нужен Python 3.10+ и `pygame-ce`:

```bash
pip install pygame-ce
python -m mvek
```

Альтернативно:

- **Windows** — двойной клик по `PLAY.bat` (запускает игру из `.venv`).
- **Linux/macOS** — `./play.sh` (сам поднимает `.venv` с `pygame-ce`)
  или `python main.py` (тонкий враппер вокруг `python -m mvek`).

> `numpy` — необязательная зависимость: с ней включаются процедурные
> звуковые эффекты, без неё игра работает и просто молчит на SFX.

#### Готовый Windows `.exe`

Можно собрать **единый файл** `MVEK.exe`, который запускается двойным
кликом без установки Python — см. раздел
[«Сборка в один Windows .exe»](#сборка-в-один-windows-exe) ниже или
файл [`BUILD_EXE.md`](BUILD_EXE.md).

### 1.2. Управление

| Клавиша | Действие |
|---|---|
| `WASD` | движение |
| Стрелки | стрелять «докладами» (twin‑stick) |
| `Space` | использовать активный предмет (если кулдаун заполнен) |
| `B` | положить хлопушку (бомбу) |
| `M` | вкл/выкл миникарту |
| `Esc` | пауза в забеге / назад в меню |
| `R` | новый забег (на экранах GAME OVER / WIN) |
| `← / →` или `A / D` | выбор персонажа / листание в меню |
| `↑ / ↓` или `W / S` | навигация по пунктам меню |
| `Tab` | переключение сложности в меню выбора персонажа |
| `Enter` / `Space` | подтвердить выбор / начать забег |
| `F11` | полноэкранный режим |
| `F10` | инструмент разметки кнопок меню (для разработки) |

> **Подбор предметов и сундуков — автоматический.** Подходишь к ним —
> срабатывают сами. Клавиша E больше не нужна.

## Меню и переходы между экранами

Игра построена как конечный автомат: всё состояние хранится в
`Game.state`, а главный цикл (`run`) на каждом кадре вызывает
`handle_events` → `update` → `draw`. Обработка ввода делегируется
методу-обработчику текущего состояния, а отрисовка — соответствующему
методу `_draw_*`.

### Состояния

| Состояние | Значение | Назначение |
|---|---|---|
| `title` | `TITLE` | Стартовый экран «НАЖМИ ДЛЯ СТАРТА» |
| `main_menu` | `MAIN_MENU` | Новая игра / Продолжить / Настройки |
| `saves` | `SAVES` | Выбор ячейки сохранения (ФАЙЛ 1/2/3) |
| `menu` | `MENU` | Выбор персонажа («кто ты?») |
| `settings` | `SETTINGS` | Громкость / экран / сложность / читы |
| `play` | `PLAY` | Игровой кадр (забег) |
| `transition` | `TRANSITION` | Слайд-анимация перехода между комнатами |
| `pause` | `PAUSE` | Пауза в забеге |
| `game_over` | `GAME_OVER` | Отчисление (HP = 0) |
| `win` | `WIN` | Диплом получен (пройден последний этаж) |

### Карта переходов

- **title → main_menu**: `ENTER` / `SPACE` или клик по зоне `start`.
  `ESC` — выход из игры.
- **main_menu**: навигация `↑↓` (`W`/`S`), выбор `ENTER`/`SPACE` или клик.
  «Новая игра» → `saves`; «Продолжить» (неактивна без сохранений) →
  загрузка слота и сразу `play`; «Настройки» → `settings`. `ESC` → `title`.
- **saves**: листание `←→` (`A`/`D`), выбор `ENTER`/`SPACE` или клик по
  карточке; `DEL`/`BACKSPACE` (или кнопка «УДАЛИТЬ ФАЙЛ») — стереть слот.
  Занятый слот → продолжить забег (`play`); пустой → `menu`. `ESC` →
  `main_menu`.
- **menu (выбор персонажа)**: листание героев `←→` (`A`/`D`) или клик по
  стрелкам `<` / `>`; `TAB` — переключение сложности NORMAL/HARD; `ENTER` —
  старт забега (`new_run` → `play`). `ESC` → `saves`.
- **settings**: выбор строки `↑↓` (`W`/`S`), изменение значения `←→`
  (`A`/`D`), применить `ENTER`/`SPACE` или клик. `ESC` — возврат туда,
  откуда вошли (в `pause`, если открыто из паузы, иначе в `main_menu`).
- **play → pause**: `ESC`. Игровые клавиши: `SPACE` — активный предмет,
  `B` — бомба, `M` — карта, `E` — ручной подбор (предметы подбираются и
  автоматически), `K` — чит «убить всех».
- **pause**: навигация `↑↓` (`W`/`S`), выбор `ENTER`/`SPACE` или клик.
  «Продолжить» → `play`; «Выход» → автосейв и `main_menu`; «Настройки» →
  `settings`. `ESC` → `play`.
- **game_over / win**: `R` — новый забег (`play`), `ESC` → `main_menu`.
  `WIN` достигается из `play` при прохождении последнего этажа;
  `GAME_OVER` — когда HP игрока ≤ 0.
- **transition**: автоматическое состояние; по истечении длительности
  (0.4 с) возвращается в `play`. Ввод не обрабатывается.

### Анимации перехода

- **Кроссфейд между экранами меню** (`_apply_fade`): при смене состояния
  снимок прежнего кадра угасает поверх нового; длительность
  `_fade_dur = 0.28` с (альфа линейно спадает к нулю).
- **Слайд между комнатами** (`TRANSITION`): обе комнаты смещаются по
  направлению перехода с easing `1 − (1 − t)³`; длительность
  `_trans_duration = 0.4` с. Запускается при касании двери в зачищённой
  комнате (`_check_door_transition`).

## Играбельные персонажи

Профили заданы в словаре `CHARACTERS` (`mvek/entities/student.py`),
порядок в меню — `CHARACTER_ORDER` (`mvek/main.py`). Поле `homing`
(автонаводка снарядов) есть только у проклятых форм. `max_love` указан в
half-hearts (как в коде); в скобках — сердца целиком (HP в меню =
`max_love // 2`).

| ID | Имя | HP (`max_love`) | СКОР (`speed`) | УРОН (`damage`) | Скорострел. (`fire_rate`) | Удача (`luck`) | Душа (`soul`) | Наводка (`homing`) | Замок / разблокировка |
|----|-----|-----------------|----------------|-----------------|---------------------------|----------------|---------------|--------------------|-----------------------|
| `platon` | ПЛАТОН | 4 (2♥) | 2.6 | 0.8 | 2.0 | 0 | 0 | нет | открыт по умолчанию — **худшие статы** |
| `kiryuha` | КИРЮХА | 8 (4♥) | 2.8 | 1.6 | 2.0 | 0 | 0 | нет | открыт |
| `nataha` | НАТАХА | 6 (3♥) | 3.4 | 0.9 | 3.0 | 1 | 0 | нет | открыт |
| `nikitos1` | НИКИТОС I | 6 (3♥) | 3.0 | 1.0 | 2.5 | 0 | 0 | нет | открыт; стартует с предметом «Энергетик "3 часа ночи"» |
| `nikitos2` | НИКИТОС II | 8 (4♥) | 3.0 | 1.1 | 2.4 | 1 | 0 | нет | открыт |
| `anka` | АНЬКА | 6 (3♥) | 3.1 | 1.0 | 2.7 | 2 | 2 | нет | открыт |
| `cursed_cupsize` | CURSED CUPSIZE ПЛАТОН | 10 (5♥) | 3.4 | 1.8 | 3.0 | 3 | 0 | **да** | `locked`; открыть 3 победами за `platon` |
| `cursed_zupsize` | CURSED ZUPSIZE ПЛАТОН | 12 (6♥) | 3.6 | 2.2 | 3.2 | 4 | 0 | **да** | `locked`; открыть 3 победами за `cursed_cupsize` |

> На сложности HARD у выбранного персонажа `max_love` урезается на 2
> (минимум 2): `Student.max_love = max(2, max_love - 2)` в `new_run`.

### Трек «ЗППП»

Песня `mvek/assets/audio/zppp.mp3` играет **только** в забеге за проклятого
Платона (`cursed_cupsize` либо `cursed_zupsize`). Логика — `_CURSED_CHARS`,
`_update_run_music`, `_music_path`, `_stop_music` в `mvek/main.py`. Для
любого обычного персонажа музыка останавливается. Трек запускается на
репите (`pygame.mixer.music.play(-1)`) и глушится при выходе в меню, на
экранах GAME OVER и WIN.

## Сохранения и прогресс

Весь прогресс лежит в одном файле `mvek_save.json` (модуль `mvek/save.py`),
устойчивом к повреждению — битый сейв молча откатывается к значениям по
умолчанию, не роняя игру. В нём четыре раздела:

- `wins` — число побед за каждого персонажа;
- `unlocked` — список открытых персонажей;
- `slots` — ровно **3** ячейки забега (ФАЙЛ 1/2/3, `None` = пусто);
- `settings` — громкость, музыка вкл/выкл, полноэкранный режим, сложность.

**Разблокировки.** После победы `record_win(char_id)` увеличивает счётчик
и открывает персонажей, чьё условие выполнено (`locked` / `unlock_by` /
`unlock_wins` в профиле). Так открываются проклятые формы Платона.

**Снимок забега (`mvek/runsave.py`).** Этаж детерминирован своим `seed`,
поэтому граф комнат не сохраняется — снимок хранит лишь `seed`, номер
этажа, посещённые/зачищенные комнаты, запертые двери и статы игрока.
`restore()` пересоздаёт этаж из seed (`Floor(level, seed)`), и карта
получается идентичной. Выход из забега через паузу делает автосейв в слот.

**Где лежит файл.** Путь выбирает `mvek/paths.py`: из исходников — в корне
репозитория, в собранном `.exe` — **рядом с `.exe`** (а не во временной
распаковочной папке, которая удаляется при выходе). Поэтому `.exe`
портативен и прогресс переживает перезапуск.

## Инструмент разметки меню (F10)

Координаты кликабельных зон и текста в меню заданы в `Game.DEFAULT_LAYOUT`
прямо в коде, а сверху накладывается необязательный `menu_layout.json`
(генерируется локально, в git не коммитится). Чтобы двигать элементы
визуально, не редактируя числа руками, есть встроенный редактор:

1. На любом экране меню нажми **`F10`** — включится режим разметки.
2. Выделяй элементы клавишами `[` / `]` (или `X` / `C`).
3. **Стрелки** двигают выделенный элемент (с `Shift` — шаг 10 px, без —
   1 px). С зажатым **`Ctrl`** стрелки меняют размер (где это
   поддерживается).
4. `,` / `.` — поворот (с `Shift` — на 15°, без — на 5°).
5. `R` — сбросить выделенный элемент к дефолту.
6. **`S`** — сохранить раскладку в `menu_layout.json`.
7. **`F10`** — выйти из режима разметки.

Каждый элемент хранит дельты `[dx, dy, dw, dh, angle]` относительно
дефолта, поэтому правки переносимы.

## Сборка в один Windows .exe

Игра собирается в **единый файл** `dist\MVEK.exe`, который запускается
двойным кликом — без установки Python и без папки с зависимостями.

> ⚠️ **Собирать можно только на Windows.** PyInstaller не умеет
> кросс-компиляцию, поэтому собрать `.exe` на Linux или macOS невозможно —
> нужна именно Windows-машина. Зато сам процесс сводится к одному клику.

### Способ 1 — в один клик (рекомендуется)

1. Скопируй всю папку проекта на Windows-машину.
2. Установи Python 3.11–3.13 с
   [python.org](https://www.python.org/downloads/), отметив галочку
   **«Add Python to PATH»**.
3. Двойной клик по **`build_exe.bat`**.

Скрипт сделает всё сам: найдёт Python, создаст отдельное окружение сборки
`.venv_build` (игровое `.venv` не трогается), поставит `pygame-ce`,
`numpy`, `pyinstaller` и запустит `pyinstaller --noconfirm --clean
MVEK.spec`. Когда появится **«ГОТОВО!»** — результат лежит в `dist\MVEK.exe`.

### Способ 2 — вручную

```bat
py -m venv .venv_build
.venv_build\Scripts\pip install pygame-ce numpy pyinstaller
.venv_build\Scripts\pyinstaller --noconfirm --clean MVEK.spec
```

### Что собирается (`MVEK.spec`)

- Режим **`--onefile`** — всё в одном `.exe`; **`console=False`** — без
  чёрного окна консоли.
- Точка входа — `main.py`; в hidden-imports добавлен `numpy` (для
  процедурных звуков).
- Вшитые ассеты (только чтение): `mvek/assets/` (иконки, спрайты, слои
  меню, музыка `zppp.mp3`) и `платон.png` (спрайт игрока).
- **UPX** включён; **своя иконка** — положи `mvek.ico` рядом с `MVEK.spec`.
- Записываемые файлы (`mvek_save.json`, `menu_layout.json`) **не**
  вшиваются — они создаются рядом с `.exe` (см. «Сохранения и прогресс»),
  поэтому `.exe` портативен.

Подробности — в [`BUILD_EXE.md`](BUILD_EXE.md).

## 3. Архитектура (короткое описание)

### 3.1. Базовая сущность — `Entity` (`mvek/ecs.py`)

Любой объект внутри комнаты наследуется от `Entity`. У него есть
`x, y, dead, solid, radius`. Метод `update(dt, room)` вызывается
каждый кадр, `draw(surface, ox, oy)` — раз за кадр для отрисовки.

Когда `entity.dead = True`, комната удалит её в конце кадра.

### 3.2. Комната — `Room` (`mvek/world/room.py`)

Комната — фиксированный 19×13 тайл-грид. Сущности живут в плоском
списке `room.entities`, снаряды — в `room.projectiles`. У комнаты
есть тип (`kind`): `start | normal | boss | treasure | shop |
secret | arcade | sacrifice | challenge | curse | miniboss`.

При зачистке (`cleared = True`) на пол падает награда: монета /
сердце / ключ / бомба / иногда сундук — детали в `_drop_reward`.

### 3.3. Этаж — `Floor` (`mvek/world/floor.py`)

Случайный граф комнат. Параметры этажа берутся из
`FLOOR_CONFIGS[level - 1]`:

```python
FLOOR_CONFIGS = [
    # (число комнат, мульт. сложности, число treasure-комнат, баннер)
    (16, 1.00, 1, "МВЭК — 1 этаж: ВЕСТИБЮЛЬ"),
    (18, 1.15, 1, "МВЭК — 2 этаж: КОРИДОРЫ"),
    (20, 1.30, 2, "МВЭК — 3 этаж: КАФЕДРЫ"),
    (22, 1.45, 2, "МВЭК — 4 этаж: ЛАБОРАТОРИИ"),
    (24, 1.60, 2, "МВЭК — 5 этаж: ДЕКАНАТ"),
]
```

После победы над боссом в boss‑комнате появляется лестница вниз
(`Stairway`); зайдя на неё, игрок попадает на следующий этаж.
После 5‑го уровня — экран WIN.

### 3.4. Игрок — `Student` (`mvek/entities/student.py`)

Имеет статы: `speed, damage, fire_rate, shot_speed, shot_range,
luck, max_love, soul, coins, keys, bombs`. Расширения механик
(orbitals, shield, freeze_tears, piercing, magnet_tears,
pickup_magnet, melee_mode, has_familiar, has_revive,
streak_speed_bonus, transformations) — отдельные поля с пометкой
«добавлено предметами».

### 3.5. Боссы и мини‑боссы

* **Финальный босс**: `Director` (`entities/boss.py`) — 2 фазы,
  4+1 паттернов атак, призывает помощников.
* **Этажные боссы 1–4**: `bosses_extra.py` — каждый со своими
  паттернами. Подключены через `make_boss_for_floor(level, x, y)`.
* **Мини‑боссы**: `minibosses.py` — 10 классов. Случайный выбор
  через `random_miniboss(rng, x, y)` для комнат типа `miniboss`.

Все они помечены атрибутом класса `is_boss = True`, что позволяет
коду снарядов/орбиталок/милишных ударов/детектора зачистки комнаты
обрабатывать их единообразно с обычными `Enemy`.

### 3.6. Предметы — `items/items.py`

Каждый предмет — словарь:

```python
{
    "name":        "Кружка кофе из автомата",  # отображаемое имя
    "kind":        "passive",                  # или "active"
    "description": "...",                      # длинное описание
    "color":       (110, 70, 50),              # цвет ауры на пьедестале
    "apply":       lambda s: setattr(s, "speed", s.speed + 1.2),
    "flavor":      "SPEED UP!",                # короткая всплывающая надпись
    "tag":         None,                       # тег для трансформации
    "icon":        ICONS["coffee_mug"],        # пиксельный спрайт
}
```

Реестр всех 55+ предметов лежит в `ITEM_REGISTRY`.
Карта `ITEMS_BY_NAME` — быстрый доступ по имени.

### 3.7. Спрайты — мини‑язык операций

Каждая иконка — список примитивов:

| Опкод | Параметры | Что рисует |
|---|---|---|
| `c` | `color, dx, dy, r` | filled-круг |
| `o` | `color, dx, dy, r, w` | outline-круг толщиной `w` |
| `r` | `color, dx, dy, w, h` | filled-прямоугольник |
| `R` | `color, dx, dy, w, h, lw` | outline-прямоугольник |
| `p` | `color, [(x, y), …]` | filled-полигон |
| `l` | `color, x1, y1, x2, y2, w` | линия |

Координаты — пиксели от центра при `scale=1.0`. Рекомендуемый
размер силуэта — ±14 px (чтобы влез в 28×28 ячейку инвентаря).

`paint_icon(surface, ops, cx, cy, scale)` отрисует список ops в
указанной точке с заданным масштабом.

### 3.8. Трансформации

Каждый предмет может иметь поле `tag`. Их 5:

| `tag` | Имя | Эффект при наборе 3 предметов |
|---|---|---|
| `honors`  | ОТЛИЧНИК | +1 урон, +0.3 скорострельности |
| `athlete` | СПОРТСМЕН | +0.4 скорости, +1 макс сердце |
| `bookworm`| БИБЛИОФИЛ | пробивающие выстрелы + фамильяр |
| `hacker`  | ХАКЕР | +2 удачи + бот-фамильяр |
| `artist`  | АРТИСТ | постоянный полёт |

Логика: после каждого `try_pickup` вызывается
`check_transformations(student)`, считает теги среди `student.passives`,
и при достижении порога 3 включает соответствующую трансформацию
(идемпотентно, через множество `student.transformations`).

## 5. Как добавить / редактировать предмет

### 5.1. Добавить пассивный предмет

1. Открой `mvek/items/items.py`.
2. Сначала **нарисуй иконку**. Добавь запись в словарь `ICONS`:

```python
ICONS["my_widget"] = [
    ('r', (80, 130, 200), 0, 0, 18, 22),   # синий корпус
    ('r', (240, 230, 200), 0, -2, 12, 8),  # экранчик
    ('c', (255, 80, 80), 0, 6, 2),         # красная лампочка
]
```

> Размер силуэта — в пределах ±14 пикселей от центра.

3. В `ITEM_REGISTRY` добавь запись:

```python
_passive(
    "Мой Виджет",                                  # name
    "+0.5 удачи, +0.3 урона.",                     # description
    (80, 130, 200),                                # color (для glow)
    apply=lambda s: (
        setattr(s, "luck", s.luck + 0.5),
        setattr(s, "damage", s.damage + 0.3),
    ),
    icon=ICONS["my_widget"],
    tag=None,             # или "honors"/"athlete"/...
    flavor="STAT UP!",    # короткая надпись на пикапе
),
```

Готово — предмет появится в пуле выпадения и в магазинах.

### 5.2. Добавить активный предмет

```python
_active(
    "Мой Активный",
    "Что делает при использовании.",
    (200, 80, 120),
    icon=ICONS["my_widget"],
    flavor="ACTIVE!",
),
```

После этого открой `mvek/entities/student.py`, метод
`Student.use_active`, и допиши ветку:

```python
if name == "Мой Активный":
    # ... твоя механика, например ...
    self.shield_t = 3.0
    self._start_cooldown(20.0)   # 20 сек кулдауна
    fx.flash(it["color"], 0.2)
    sounds.play("phase")
    return True
```

`_start_cooldown(seconds)` обновит и `berserk_cd` (текущий
оставшийся кулдаун), и `berserk_cd_max` (длительность для шкалы).

### 5.3. Изменить существующий предмет

Найди его по имени в `ITEM_REGISTRY`. Поменяй `apply`,
`description`, `color`, `icon` или `tag`. Изменения подхватятся
при перезапуске.

### 5.4. Привязать к трансформации

В записи предмета поставь `tag="honors"` (или другой ключ из
`TRANSFORMATIONS`). Когда у игрока в `passives` окажется ≥ 3
предмета с этим тегом, сработает `_apply_transformation` и
включится постоянный бонус.

Чтобы добавить **новую** трансформацию:
1. В `TRANSFORMATIONS` добавь запись:
   ```python
   "engineer": {
       "name": "ИНЖЕНЕР",
       "threshold": 3,
       "flavor": "+0.5 урона, +0.5 fire-rate.",
       "color": (180, 200, 220),
   },
   ```
2. В `_apply_transformation` допиши ветку с эффектом.
3. Пометь нужные предметы `tag="engineer"`.

## 6. Как добавить нового босса (этажного)

1. Открой `mvek/entities/bosses_extra.py`.
2. Создай класс‑наследник `_BossBase`:

```python
class HeadOfDormitory(_BossBase):
    """Босс этажа 6 — призывает 3 студентов и стреляет ключами."""

    NAME = "ЗАВ. ОБЩЕЖИТИЕМ"
    HP   = 150
    RADIUS = 28

    def __init__(self, x, y):
        super().__init__(x, y)
        self._fire_cd = 1.0

    def update(self, dt, room):
        from mvek.entities.student import Student
        player = next((e for e in room.entities
                       if isinstance(e, Student)), None)
        if player is None:
            return
        # ... твои паттерны атак ...
        self._contact_damage(room)

    def draw(self, surface, ox, oy):
        # ... отрисуй спрайт ...
        self._draw_hp_bar(surface, ox, oy)
```

3. Добавь его в `make_boss_for_floor`:

```python
if level == 6:
    return HeadOfDormitory(x, y)
```

4. Если нужен новый этаж — добавь запись в `FLOOR_CONFIGS` в
   `mvek/world/floor.py`.

> Базовый класс уже даёт: HP, hit-flash, контактный урон,
> верхний HP‑бар. Тебе остаётся только `update` и `draw`.

## 9. Тонкие моменты / gotchas

* **Авто‑подбор**. `Game._try_pickup()` дёргается каждый кадр в
  `Game.update`. Если ты сделаешь сундук, который не должен
  открываться сам, ставь у него `auto_open = False` и проверь
  это в `_try_pickup`.
* **`is_boss`**. Любой враг, у которого этого флага нет,
  обрабатывается как обычный `Enemy`. Если новый босс не реагирует
  на снаряды — проверь, что `is_boss = True` в его классе.
* **Трансформации идемпотентны.** Чтобы перезапустить эффект —
  очисти `student.transformations` и снова вызови
  `check_transformations`.
* **Цвета** в `pygame.draw.*` должны быть валидными RGB‑тройками
  без `None`. Если предмет рисуется белым квадратом — почти
  наверняка в `icon` опечатка в опкоде.
* **Снаряды‑папки боссов** наследуют `_is_flask` или `piercing`
  через присваивание атрибутов (см. `LabHead._fire`). Это
  утиная типизация, без декларации в классе `Projectile`.
* **`_BossBase.take_damage`** имеет «soft armor», который
  ограничивает урон одной пули. Если тестируешь босса с D6/D8
  и хочешь чистые цифры — временно поставь
  `eff = dmg` в начале метода.
* **`student._last_room`** ставится в `Student.update`. Активные
  предметы, которые спавнят сущности (`Алебастровая шкатулка`,
  `Откровение деканата`), читают это поле — если их позвать до
  первого кадра, они тихо ничего не сделают.

## 10. Известные «ручки баланса»

* **Сложность этажа** регулируется первой цифрой в `FLOOR_CONFIGS`
  (число комнат) и второй (`diff_mult`, множитель числа врагов в
  `_populate`).
* **Скорость роста урона** игрока — в основном через
  `PLAYER_BASE_DAMAGE` и предметы с `apply` вида `s.damage += N`.
* **Soft‑armor боссов** — формула в `_BossBase.take_damage`:
  `dmg / (1 + 0.4*(dmg-1))^0.6`. Подкручивай показатели степени,
  если бой ощущается слишком быстрым / медленным.
* **Кулдауны активных предметов** настраиваются вызовом
  `_start_cooldown(seconds)` в `Student.use_active`.

## 13. Контакты / соглашения

* Названия классов — `CamelCase`, функций и модулей —
  `snake_case`.
* Игровые имена в строках — на русском, имена символов кода — на
  английском.
* Семейный формат: никакой крови / трэша / неуместных шуток в
  коде, строках и спрайтах.
* `pygame-ce` — единственная обязательная зависимость; `numpy`
  (процедурные звуки) и `pyinstaller` (сборка `.exe`) — опциональны.
  Всё остальное на чистом стандартном Python.
